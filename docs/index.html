About This Database ▼

Pre-computed multigroup, angle-binned Compton scattering cross sections evaluated using the Kershaw–Prasad–Beason (1986) thermal Compton kernel via the ComptonMatrixExact deterministic solver with uniform energy weighting.

The multigroup, angle-binned scattering cross section is defined as:

$$\sigma\bigl(g \to g',\,[\xi_{i-\frac{1}{2}},\,\xi_{i+\frac{1}{2}}];\,T\bigr) = \frac{2\pi \displaystyle\int_{\mathcal{E}_{g-\frac{1}{2}}}^{\mathcal{E}_{g+\frac{1}{2}}} \int_{\mathcal{E}'_{g'-\frac{1}{2}}}^{\mathcal{E}'_{g'+\frac{1}{2}}} \int_{\xi_{i-\frac{1}{2}}}^{\xi_{i+\frac{1}{2}}} \mathcal{K}(\mathcal{E},\,\mathcal{E}',\,\xi;\,T)\;d\xi\;d\mathcal{E}'\;d\mathcal{E}} {\mathcal{E}_{g+\frac{1}{2}} - \mathcal{E}_{g-\frac{1}{2}}}$$

where \(\mathcal{K}(\mathcal{E},\,\mathcal{E}',\,\xi;\,T)\) is the thermal Compton scattering kernel, \(\mathcal{E}\) and \(\mathcal{E}'\) are the incoming and outgoing photon energies, \(\xi = \cos\theta\) is the cosine of the scattering angle, \(T\) is the electron temperature, and the half-integer subscripts denote the group and bin boundaries.

Accuracy Notes ▼

The data is stored on a fine grid of 128 log-spaced energy groups and 100 uniform angle bins. Collapsing to a coarser grid is exact when the requested boundaries coincide with the stored fine-grid boundaries.

When custom energy boundaries fall between fine-grid points, the code splits the affected fine groups proportionally by width (linear interpolation). This introduces a few-percent error on the dominant matrix elements, typically ≤ 5–6% relative error.

Since the angle bin resolution is finer than the energy group resolution, custom angle grids have less interpolation error than custom energy grids.

Temperature interpolation: When a custom temperature is specified that does not match a stored grid point, the matrix is computed by linear interpolation in log-temperature space between the two nearest stored temperatures. The 64 stored temperatures are log-spaced, so adjacent points have a ratio of approximately 1.25.

For the best accuracy, use the auto-generated log-spaced boundaries and select stored temperature values from the dropdown.

What is .npy? ▼

.npy is NumPy's native single-array binary format.

Load in Python:

import numpy as np

# Multiple temperatures, energy groups, and angle bins → 4D array
sigma = np.load("compton_sigma_3T_16g_10a.npy")
print(sigma.shape)   # (3, 16, 16, 10)
# sigma[temperature, g_in, g_out, angle_bin]

# Single temperature → falls back to 3D array
sigma = np.load("compton_sigma_T053_16g_10a.npy")
print(sigma.shape)   # (16, 16, 10)
# sigma[g_in, g_out, angle_bin]

# 1 angle bin, single temperature → 2D matrix
sigma = np.load("compton_sigma_T053_16g_1a.npy")
print(sigma.shape)   # (16, 16)
# sigma[g_in, g_out]

# 1 energy group, single temperature → 1D angle array
sigma = np.load("compton_sigma_T053_1g_10a.npy")
print(sigma.shape)   # (10,)
# sigma[angle_bin]

# 1 energy group, 1 angle bin, single temperature → scalar
sigma = np.load("compton_sigma_T053_1g_1a.npy")
print(sigma.shape)   # ()
print(float(sigma))

# Load metadata arrays
temps = np.load("temperatures.npy")          # (nT,) in Kelvin
energies = np.load("energy_boundaries_keV.npy")  # (nG+1,)
angles = np.load("angle_boundaries_xi.npy")     # (nA+1,)
